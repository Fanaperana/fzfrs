# Security Policy

## Supported Versions

This project is currently maintained on the latest state of the default branch.
Security fixes, when needed, will be applied there first.

## Reporting a Vulnerability

Please do not open a public issue for security-sensitive reports.

Instead, use one of these options:

- Open a private GitHub security advisory for this repository
- Contact the maintainer privately through GitHub

When reporting, include:

- A clear description of the issue
- Steps to reproduce
- Impact assessment
- Any proof of concept or suggested fix, if available

You can expect an acknowledgment after the report is reviewed. Valid reports
will be investigated and fixed as appropriate.
